# Task Manager REST API

Масштабируемый RESTful API сервис для управления задачами с системой пользователей, категорий и приоритезации.

<br>

### Стек технологий: 
- Язык: Python 3.10+
- Фреймворк: FastAPI 
- База данных: PostgreSQL
- Тестирование: Postman
- Контейнеризация: Docker
- Безопасность: JWT (python-jose), Хеширование паролей (passlib, bcrypt)
- Допольнительно: Pydantic (Валидация), Psycopg2 (работа с БД, нативные sql запросы)

<br>

### Архитектура и дизайн:
- RESTful API с правильной семантикой HTTP методов (GET, POST, PUT, PATCH, DELETE)
- Реляционная модель данных с внешними ключами и каскадными операциями
- Многоуровневая валидация данных на уровне API и базы данных
- Обработка ошибок с корректными HTTP статус кодами (400, 404, 500)
- Защита от SQL-инъекций через параметризованные запросы

### Безопасность и Аутентификация
- Регистрация пользователей с хешированием паролей (алгоритм **bcrypt**).
- Выдача **JWT-токенов** при логине для безопасной сессии.
- Защита приватных эндпоинтов: пользователь может создавать, редактировать и удалять **только свои задачи**.
- Возврат ```403 Forbidden``` при попытке изменить чужие ресурсы.

<br>

### Функциональность:
- CRUD операции для задач, пользователей и категорий
- Фильтрация и выборка данных:
  - Получение задач по категориям
  - Получение задач по пользователям
  - Сортировка по дате создания
- Система приоритетов и оценка временных затрат
- Отслеживание статуса выполнения задач
- Аудит изменений (автоматическое обновление ```updated_at``` через триггеры БД)

<br>

### Как запустить проект

1) Запуск базы данных

Если у вас ещё не запущен контейнер с БД:
```docker run --name my_postgres -e POSTGRES_PASSWORD=mysecretpassword -e POSTGRES_DB=taskdb -p 5433:5432 -d postgres```

2) Сборка и запуск API

Соберите Docker образ:
```docker build -t task-api .```
Запустите контейнер, передав адрес хоста для подключения к БД:
```docker run -d -p 8000:8000 -e DB_HOST=host.docker.internal --name my_task_api task-api```

3) Проверка работы:

Откройте браузер и перейдите по адресу:
```http://localhost:8000/docs```
Откроется документация Swagger UI где можно протестировать все эндпоинты

После создания этих двух контейнеров, при следующем запуске вашего ПК, запустить эти контейнеры можно двумя командами

```docker start my_postgres```

```docker start my_task_api```

<br>

### Тестирование

В репозиторий добавлена коллекция запросов для Postman (TaskManager.postman_collection.json).
Она включает в себя:
- Полный цикл аутентификации (Регистрация → Логин → Получение JWT)
- Сценарии "Happy Path" (создание, чтение, обновление, удаление).
- Тесты на обработку ошибок (401 Unauthorized, 403 Forbidden, 404 Not Found).
- Проверку каскадного удаления задач при удалении пользователя.
Вы можете импортировать этот файл в свой Postman и сразу начать тестировать API.

### ENDPOINTS

Аутентификация:
- POST /register — Регистрация нового пользователя
- POST /login — Логин и получение JWT токена

Пользователи:
- GET /users — Список всех пользователей
- GET /users/me — Данные текущего пользователя (требует токен)
- GET /users/me/tasks — Задачи текущего пользователя (требует токен)
- DELETE /users/me — Удаление текущего пользователя и его задач (требует токен)

Задачи:
- GET /tasks — Получить все задачи (с фильтрацией по category_id)
- POST /tasks — Создать новую задачу (требует токен, user_id берётся из токена)
- PUT /tasks/{task_id} — Полностью обновить задачу (требует токен)
- PATCH /tasks/{task_id}/done — Отметить задачу выполненной (требует токен)
- DELETE /tasks/{task_id} — Удалить задачу (требует токен)

Категории:
- GET /categories — Список всех категорий
