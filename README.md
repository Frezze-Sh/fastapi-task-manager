# fastapi
Task Management REST API

Масштабируемая RESTful API сервис для управления задачами с системой пользователей, категорий и приоритезации.

<br>

Технологии: 
- FastAPI 
- PostgreSQL
- Pydantic
- Docker
- ORM: Psycopg2 (нативные sql запросы)

<br>

Архитектура и дизайн:
- RESTful API с правильной семантикой HTTP методов (GET, POST, PUT, PATCH, DELETE)
- Реляционная модель данных с внешними ключами и каскадными операциями
- Многоуровневая валидация данных на уровне API и базы данных
- Обработка ошибок с корректными HTTP статус кодами (400, 404, 500)
- JSON Request/Response

<br>

Функциональность:
- CRUD операции для задач, пользователей и категорий
- Фильтрация и выборка данных:
  - Получение задач по категориям
  - Получение задач по пользователям
  - Сортировка по дате создания
- Система приоритетов и оценка временных затрат
- Отслеживание статуса выполнения задач
- Аудит изменений (created_at, updated_at timestamps)

<br>

Категоризация:
- Иерархическая организация задач
- Настраиваемые категории с метаданными (цвет, иконки)
- Внешние ключи с каскадным удалением

<br>

ENDPOINTS

Tasks:
- GET /tasks - получить все задачи (с фильтрацией по category_id)
- GET /tasks/{task_id} - получить задачу по ID
- POST /tasks - создать новую задачу
- PUT /tasks/{task_id} - полностью обновить задачу
- PATCH /tasks/{task_id}/done - отметить задачу выполненной
- DELETE /tasks/{task_id} - удалить задачу

Users:
- GET /users - список всех пользователей
- POST /users - создать пользователя
- GET /users/{user_id}/tasks - задачи конкретного пользователя

Categories:
- GET /categories - список всех категорий
